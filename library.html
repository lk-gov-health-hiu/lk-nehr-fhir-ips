The IPS for Sri Lanka is a document. All the profiles specified by this guide are used by the Sri Lanka IPS Composition and are derived from the base IPS specification itself.

For the full list of system generated artifacts for this implementation guide, see Artifacts Summary.

  • Example implementation for each of these profiles can be seen in the Examples tab of the profile definition.

HIMS

IPS Bundle
This profile represents the constraints applied to the Bundle resource that is specific to HIMS but derived from the IPS FHIR Implementation Guide.

Transactional Bundle
This bundle contains all of the HIMS resources for transactional requests.

Composition
This profile represents the International Patient Summary (IPS) data set for HIMS.

Risk Assessment - Cardiovascular Disease
This profile represents the patient's risk for cardiovascular disease.

Service Request - Referral Request For Follow-up Plan
This profile represents the referral request for a Follow-up Plan.

CarePlan - Follow-up At HLC
This profile represents the follow-up event for the patient at HLC.

Condition - Medical History
This profile represents previous, pre-existing and new conditions for the patient.

Observation - Random Blood Sugar
This profile represents the patient's RBS results.

Observation - Fasting Blood Sugar
This profile represents the patient's FBS results.

Observation - Blood Pressure
This profile represents the patient's blood pressure.

Observation - Total Cholesterol
This profile represents the patient's total cholesterol results.

Observation - Physical Activity Status
This profile represents the physical activity status of the patient.

Observation - Tobacco Smoker
This profile represents the tobacco smoking status of the patient.

Observation - Patient Height
This profile represents the patient's height.

Observation - BMI
This profile represents the patient's BMI.

Allergy Intolerance - No Allergy Information Available
This profile indicates that there is no allergy information available for the patient.

Medication Request - No Prescriptions
This profile represents the fact that the patient has not received any prescriptions for drugs.

HHIMS

IPS Bundle
This profile represents the constraints applied to the Bundle resource that is specific to HHIMS but derived from the IPS FHIR Implementation Guide.

Transactional Bundle
This bundle contains all of the HHIMS resources for transactional requests.

Composition
This profile represents the International Patient Summary (IPS) data set for HHIMS.

Medication Dispense
This profile represents dispensed medication for an HHIMS patient.

Communication - Notifiable Diseases Notified
This profile represents a message communicated to the practitioner about a patient's encounter.

Imaging Study
This profile represents the content or results of a imaging study.

Condition - Medical History Not Available
This profile indicates that there is no information available about the subject's health problems or disabilities.

Allergy Intolerance
This profile represents the allergy information available for the patient.

Medication Request
This profile is to record requests for medication that are prescribed to a patient or for Non-Medical Supplies Division (MDS) medicines.

Service Request - Procedures
This profile initiates a request for a procedure.

Service Request - Investigations
This profile initiates a request for an investigation.

Service Request - Imaging
This profile initiates a request for imaging to be done.

Task - Referral Task
This profile is used to track the progress of a patient's referral.

Shared

Patient
This profile represents demographics and other administrative information about an HIMS or HHIMS individual receiving care or other health-related services.

Encounter
This profile represents the current facility at which the HIMS or HHIMS patient is receiving health services.

Practitioner
This profile represents the practitioners who participated in the observation and/or procedure.

Observation - Patient Weight
This profile represents the patient's weight.

Location
This profile represents the physical location of the provider.

Organization
This profile represents the organization providing health related services.

Device Information
This profile is used to record key information about the system that contributed to the patient’s record.
